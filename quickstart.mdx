# Quickstart

This guide walks you through a complete data pipeline: loading a CSV, running vectorized math, inspecting results, and writing to a relational database — all at native Rust speed.

<Steps>
  <Step title="Load data from CSV">
    PardoX spawns a Rust thread pool to parse chunks of the CSV file in parallel. No Python objects are created during ingestion — data flows directly into Rust-managed memory buffers.

    <CodeGroup>

    ```python Python
    import pardox as px

    # Load data — parallel Rust CSV parser, automatic type inference
    df = px.read_csv("sales_data.csv")
    print(f"Loaded {df.shape[0]:,} rows × {df.shape[1]} columns")

    # Inspect columns
    print(df.columns)
    df.show(5)  # ASCII table preview
    ```

    ```javascript JavaScript
    const { read_csv } = require('pardox');

    // Load data — parallel Rust CSV parser
    const df = read_csv('./sales_data.csv');
    console.log(`Loaded ${df.shape[0].toLocaleString()} rows × ${df.shape[1]} columns`);

    // Inspect columns
    console.log(df.columns);
    df.show(5);  // ASCII table preview
    ```

    ```php PHP
    <?php
    require_once 'vendor/autoload.php';

    use PardoX\DataFrame;

    // Load data — parallel Rust CSV parser
    $df = DataFrame::read_csv('sales_data.csv');
    echo "Loaded " . number_format($df->shape[0]) . " rows × {$df->shape[1]} columns\n";

    // Inspect columns
    print_r($df->columns);
    echo $df->show(5);  // ASCII table preview
    ```

    </CodeGroup>

    <Info>
      **Type inference:** The engine scans the first rows to classify each column as `Int64`, `Float64`, or `Utf8` (string). You can override with an explicit schema:

      <CodeGroup>
      ```python Python
      df = px.read_csv("data.csv", schema={"price": "Float64", "id": "Int64"})
      ```

      ```javascript JavaScript
      const df = read_csv('./data.csv', { price: 'Float64', id: 'Int64' });
      ```

      ```php PHP
      $df = DataFrame::read_csv('data.csv', ['price' => 'Float64', 'id' => 'Int64']);
      ```
      </CodeGroup>
    </Info>
  </Step>

  <Step title="Inspect your data">
    Use built-in methods to understand your dataset:

    <CodeGroup>

    ```python Python
    print(df.shape)      # (50000, 14)
    print(df.columns)    # ['id', 'price', 'quantity', ...]
    print(df.dtypes)     # {'id': 'Utf8', 'price': 'Float64', ...}
    df.show(5)           # ASCII table, first 5 rows
    ```

    ```javascript JavaScript
    console.log(df.shape);      // [50000, 14]
    console.log(df.columns);    // ['id', 'price', 'quantity', ...]
    console.log(df.dtypes);     // { id: 'Utf8', price: 'Float64', ... }
    df.show(5);                 // ASCII table, first 5 rows
    ```

    ```php PHP
    print_r($df->shape);      // [50000, 14]
    print_r($df->columns);    // ['id', 'price', 'quantity', ...]
    print_r($df->dtypes);     // ['id' => 'Utf8', 'price' => 'Float64', ...]
    echo $df->show(5);        // ASCII table, first 5 rows
    ```

    </CodeGroup>
  </Step>

  <Step title="Cast types and compute">
    When a column is inferred as `Int64` but you need `Float64` for arithmetic, cast it first:

    <CodeGroup>

    ```python Python
    # Cast quantity to Float64
    df.cast("quantity", "Float64")

    # Compute revenue = price × quantity
    revenue_df = df.mul("price", "quantity")  # new DataFrame with 'result_mul'

    # Statistics
    std_val = revenue_df.std("result_mul")
    print(f"Revenue std dev: {std_val:,.2f}")
    ```

    ```javascript JavaScript
    // Cast quantity to Float64
    df.cast('quantity', 'Float64');

    // Compute revenue = price × quantity
    const revenueDf = df.mul('price', 'quantity');  // new DataFrame with 'result_mul'

    // Statistics
    const stdVal = revenueDf.std('result_mul');
    console.log(`Revenue std dev: ${stdVal.toFixed(2)}`);
    ```

    ```php PHP
    // Cast quantity to Float64
    $df->cast('quantity', 'Float64');

    // Compute revenue = price × quantity
    $revenueDf = $df->mul('price', 'quantity');  // new DataFrame with 'result_mul'

    // Statistics
    $stdVal = $revenueDf->std('result_mul');
    echo "Revenue std dev: " . number_format($stdVal, 2) . "\n";
    ```

    </CodeGroup>

    <Note>
      Supported types: `Int64`, `Float64`, `Utf8`
    </Note>
  </Step>

  <Step title="Analyze with Observer methods">
    Use Observer methods to explore your data:

    <CodeGroup>

    ```python Python
    # Value frequency table
    state_counts = df.value_counts("state")
    print(state_counts)  # {'TX': 6345, 'CA': 6301, ...}

    # Unique values in a column
    unique_cats = df.unique("category")

    # Export full DataFrame to Python
    records = df.to_dict()   # list of dicts — 50k records
    json_str = df.to_json()  # JSON string "[{...}, ...]"
    ```

    ```javascript JavaScript
    // Value frequency table
    const stateCounts = df.valueCounts('state');
    console.log(stateCounts);  // { TX: 6345, CA: 6301, ... }

    // Unique values in a column
    const uniqueCats = df.unique('category');

    // Export full DataFrame to JavaScript
    const records = df.toDict();   // array of objects
    const jsonStr = df.toJson();   // JSON string
    ```

    ```php PHP
    // Value frequency table
    $stateCounts = $df->value_counts('state');
    print_r($stateCounts);  // ['TX' => 6345, 'CA' => 6301, ...]

    // Unique values in a column
    $uniqueCats = $df->unique('category');

    // Export full DataFrame to PHP
    $records = $df->to_dict();   // array of associative arrays
    $jsonStr = $df->to_json();   // JSON string
    ```

    </CodeGroup>

    <Tip>
      These methods use proper heap memory management for large datasets, making them safe for DataFrames with millions of rows.
    </Tip>
  </Step>

  <Step title="Write to database">
    PardoX connects to PostgreSQL, MySQL, SQL Server, and MongoDB entirely through the Rust core — no additional drivers needed.

    <CodeGroup>

    ```python Python
    from pardox.io import execute_sql

    CONN = "postgresql://user:password@localhost:5432/mydb"

    # Create table
    execute_sql(
        CONN,
        "CREATE TABLE IF NOT EXISTS sales (price FLOAT, quantity FLOAT)"
    )

    # Write data (auto COPY FROM STDIN for > 10k rows)
    rows = df.to_sql(CONN, "sales", mode="append")
    print(f"Written {rows:,} rows to PostgreSQL")
    ```

    ```javascript JavaScript
    const { executeSql } = require('pardox');

    const CONN = 'postgresql://user:password@localhost:5432/mydb';

    // Create table
    executeSql(
        CONN,
        'CREATE TABLE IF NOT EXISTS sales (price DOUBLE PRECISION, quantity DOUBLE PRECISION)'
    );

    // Write data (auto COPY FROM STDIN for > 10k rows)
    const rows = df.toSql(CONN, 'sales', 'append');
    console.log(`Written ${rows.toLocaleString()} rows to PostgreSQL`);
    ```

    ```php PHP
    use PardoX\IO;

    $conn = 'postgresql://user:password@localhost:5432/mydb';

    // Create table
    IO::executeSql(
        $conn,
        'CREATE TABLE IF NOT EXISTS sales (price DOUBLE PRECISION, quantity DOUBLE PRECISION)'
    );

    // Write data (auto COPY FROM STDIN for > 10k rows)
    $rows = $df->to_sql($conn, 'sales', 'append');
    echo "Written " . number_format($rows) . " rows to PostgreSQL\n";
    ```

    </CodeGroup>

    <Info>
      **Auto-optimization:** For PostgreSQL, PardoX automatically activates `COPY FROM STDIN` for bulk inserts when writing more than 10,000 rows. For MySQL, it uses chunked batch `INSERT` statements.
    </Info>
  </Step>

  <Step title="Persist to native format">
    Save your processed data in PardoX's native `.prdx` binary format for blazing-fast reloads:

    <CodeGroup>

    ```python Python
    # Save
    df.to_prdx("sales_processed.prdx")

    # Load (4.6 GB/s read throughput)
    df2 = px.read_prdx("sales_processed.prdx")
    ```

    ```javascript JavaScript
    const { read_prdx } = require('pardox');

    // Save
    df.toPrdx('./sales_processed.prdx');

    // Load (4.6 GB/s read throughput)
    const df2 = read_prdx('./sales_processed.prdx');
    ```

    ```php PHP
    use PardoX\IO;

    // Save
    $df->to_prdx('sales_processed.prdx');

    // Load (4.6 GB/s read throughput)
    $data = IO::read_prdx('sales_processed.prdx');
    $df2 = new DataFrame($data);
    ```

    </CodeGroup>

    <Note>
      **Performance benchmark:** Loading 2 GB of data:
      - CSV: ~8 seconds
      - Parquet: ~3 seconds
      - **PRDX: ~0.5 seconds**
    </Note>
  </Step>
</Steps>

## Full pipeline example

Here's a complete ETL pipeline that demonstrates all the features:

<CodeGroup>

```python Python
import pardox as px
from pardox.io import execute_sql

CONN = "postgresql://pardox:secret@localhost:5432/analytics"

# Load
df = px.read_csv("sales_50k.csv")
df.cast("quantity", "Float64")

# Transform
df.fillna(0.0)
df['revenue'] = df['price'] * df['quantity']
df['tax'] = df['revenue'] * 0.08

# Analyze
print(f"Total revenue : ${df['revenue'].sum():,.2f}")
print(f"Avg ticket    : ${df['revenue'].mean():,.2f}")
print(f"Std deviation : {df['revenue'].std():,.2f}")

# Inspect
top_states = df.value_counts("state")
print("Top states:", list(top_states.items())[:5])

# Write
execute_sql(CONN, "DROP TABLE IF EXISTS sales_results")
execute_sql(CONN, (
    "CREATE TABLE sales_results "
    "(price FLOAT, quantity FLOAT, revenue FLOAT, tax FLOAT)"
))
rows = df.to_sql(CONN, "sales_results", mode="append")
print(f"\nWrote {rows:,} rows to PostgreSQL")

# Save locally
df.to_prdx("sales_results.prdx")
print("Saved to sales_results.prdx")
```

```javascript JavaScript
const { read_csv, executeSql } = require('pardox');

const CONN = 'postgresql://pardox:secret@localhost:5432/analytics';

// Load
const df = read_csv('./sales_50k.csv');
df.cast('quantity', 'Float64');

// Transform
df.fillna(0.0);
df['revenue'] = df['price'].mul(df['quantity']);
df['tax'] = df['revenue'].mul(0.08);

// Analyze
console.log(`Total revenue : $${df['revenue'].sum().toLocaleString()}`);
console.log(`Avg ticket    : $${df['revenue'].mean().toFixed(2)}`);
console.log(`Std deviation : ${df['revenue'].std().toFixed(2)}`);

// Inspect
const topStates = df.valueCounts('state');
console.log('Top states:', Object.entries(topStates).slice(0, 5));

// Write
executeSql(CONN, 'DROP TABLE IF EXISTS sales_results');
executeSql(CONN,
    'CREATE TABLE sales_results ' +
    '(price DOUBLE PRECISION, quantity DOUBLE PRECISION, revenue DOUBLE PRECISION, tax DOUBLE PRECISION)'
);
const rows = df.toSql(CONN, 'sales_results', 'append');
console.log(`\nWrote ${rows.toLocaleString()} rows to PostgreSQL`);

// Save locally
df.toPrdx('./sales_results.prdx');
console.log('Saved to sales_results.prdx');
```

```php PHP
<?php
require_once 'vendor/autoload.php';

use PardoX\DataFrame;
use PardoX\IO;

$conn = 'postgresql://pardox:secret@localhost:5432/analytics';

// Load
$df = DataFrame::read_csv('sales_50k.csv');
$df->cast('quantity', 'Float64');

// Transform
$df->fillna(0.0);
$df['revenue'] = $df['price']->mul($df['quantity']);
$df['tax'] = $df['revenue']->mul(0.08);

// Analyze
echo "Total revenue : $" . number_format($df['revenue']->sum(), 2) . "\n";
echo "Avg ticket    : $" . number_format($df['revenue']->mean(), 2) . "\n";
echo "Std deviation : " . number_format($df['revenue']->std(), 2) . "\n";

// Inspect
$topStates = $df->value_counts('state');
echo "Top states: ";
print_r(array_slice($topStates, 0, 5, true));

// Write
IO::executeSql($conn, 'DROP TABLE IF EXISTS sales_results');
IO::executeSql($conn,
    'CREATE TABLE sales_results ' .
    '(price DOUBLE PRECISION, quantity DOUBLE PRECISION, revenue DOUBLE PRECISION, tax DOUBLE PRECISION)'
);
$rows = $df->to_sql($conn, 'sales_results', 'append');
echo "\nWrote " . number_format($rows) . " rows to PostgreSQL\n";

// Save locally
$df->to_prdx('sales_results.prdx');
echo "Saved to sales_results.prdx\n";
```

</CodeGroup>

## Advanced features

<CardGroup cols={2}>
  <Card title="GPU acceleration" icon="microchip">
    Use WebGPU Bitonic sort for massive datasets:

    <CodeGroup>
    ```python Python
    sorted_df = df.sort_values("price", ascending=True, gpu=True)
    ```

    ```javascript JavaScript
    const sortedDf = df.sortValues('price', true, true);
    ```

    ```php PHP
    $sortedDf = $df->sortValues('price', true, true);
    ```
    </CodeGroup>
  </Card>

  <Card title="NumPy integration" icon="arrows-split-up-and-left">
    Zero-copy bridge to NumPy (Python only):

    ```python
    import numpy as np

    # Direct pointer from Rust buffer — no data copy
    arr = np.array(df["price"])
    print(arr.dtype)   # float64
    print(arr.mean())  # same value as df["price"].mean()
    ```
  </Card>

  <Card title="Multiple databases" icon="database">
    Connect to different databases in the same pipeline:

    <CodeGroup>
    ```python Python
    # Read from PostgreSQL
    df_pg = px.read_sql(PG_CONN, "SELECT * FROM orders")

    # Write to MySQL
    df_pg.to_mysql(MYSQL_CONN, "orders", mode="append")
    ```

    ```javascript JavaScript
    const { read_sql } = require('pardox');

    // Read from PostgreSQL
    const dfPg = read_sql(PG_CONN, 'SELECT * FROM orders');

    // Write to MySQL
    dfPg.toMysql(MYSQL_CONN, 'orders', 'append');
    ```
    </CodeGroup>
  </Card>

  <Card title="Min-Max normalization" icon="chart-line">
    Normalize columns to [0, 1] range:

    <CodeGroup>
    ```python Python
    normed_df = df.min_max_scale("price")
    # Result in 'result_minmax' column
    ```

    ```javascript JavaScript
    const normedDf = df.minMaxScale('price');
    ```

    ```php PHP
    $normedDf = $df->minMaxScale('price');
    ```
    </CodeGroup>
  </Card>
